## Phase 1: Database & Core Contract Design (Days 1–5)

### Day 1: PostgreSQL Setup and Table Architecture

* **Task 1.1:** Install PostgreSQL locally using your preferred method (Docker, Homebrew, or the official installer).
* **Task 1.2:** Initialize a new database cluster and create a database named `nexus_fabric`. Open your SQL client (pgAdmin, DBeaver, or terminal psql).
* **Task 1.3:** Create the `intents` table to log the incoming text prompt from the user. Include columns for unique ID, raw string, status tracking strings (`PENDING`, `PLANNING`, `COMPLETED`), and automated timestamps.
* **Task 1.4:** Create the `execution_steps` table. Establish a explicit foreign key relation back to `intents(id)`. Add numerical columns for sorting order, text names for downstream targets, generic JSONB payload storage, and error string limits.
* **Task 1.5:** Create the `audit_logs` security table. Make sure it stores historical entries detailing exact moments of authorization, execution failures, or manual overrides.
* **Task 1.6:** Run dummy SQL `INSERT` statements manually on all three tables to guarantee constraints, keys, and data types accept records perfectly.
